# UI管理器模块拆分重构计划

## 目标
将 `UIMgr` 中的三个 partial class（UIMgrRes、UIMgrStack、UIMgrBlur）拆分为独立的 Handler 模块，保持功能不变的同时实现更好的代码解耦和单一职责原则。

---

## 当前结构分析

### 1. UIMgrRes 模块
- **职责**：UI资源加载/卸载（FairyGUI包、纹理）
- **公开方法**：`LoaUIdPackage(string[])`, `RemoveUIPackage(string[])`
- **依赖**：YooAsset, YooPackage, FairyGUI.UIPackage, UIPkgRef

### 2. UIMgrStack 模块
- **职责**：UI界面栈管理（Stack类型界面的进栈/退栈）
- **关键数据**：`_uiStacks`, `_backStacks`, `_showed`, `_showing`, `_createdDels`
- **关键方法**：`_ClearStack()`, `_ShowedPushStack()`, `_HideBeforePopStack()`, `_ShowByStack()`, `_HideByStack()`
- **依赖UIMgr**：调用 `_ShowAsync` 方法显示界面

### 3. UIMgrBlur 模块
- **职责**：UI模糊效果管理
- **关键数据**：`_blurStacks`, `_showed`, `_mainCamera`
- **关键方法**：`_ShowCallBack_Blur()`, `_HideCallBack_Blur()`, `_FlagBlur()`, `SetSceneCamera()`
- **依赖UIMgr**：访问 `_showed` 字典设置 Filter

---

## 模块间依赖关系

```
UIMgr
├── UIResourceHandler (无外部依赖，独立)
│   └── 职责: 资源管理
│
├── UIStackHandler (依赖UIMgr的内部状态和回调)
│   ├── 依赖: _showed (只读), _showing (读写), _createdDels (读写)
│   ├── 回调: 请求UIMgr显示界面 (IUIMgrStackHandlerCallback)
│   └── 职责: 栈管理
│
└── UIBlurHandler (依赖UIMgr的内部状态)
    ├── 依赖: _showed (读写Filter), _mainCamera (读写)
    ├── 回调: 无（直接访问showed）
    └── 职责: 模糊效果
```

## 重构方案

### 1. 创建接口和回调定义

新建文件 `UIHandlerInterfaces.cs`：

```csharp
namespace Ux
{
    // UIStackHandler回调接口
    public interface IUIMgrStackHandlerCallback
    {
        // 请求显示栈中的界面
        void ShowByStack(int id, IUIParam param);
        // 隐藏栈中的界面
        void HideByStack(int id);
        // 获取当前正在显示的UI列表（用于判断界面状态）
        bool IsShowing(int id);
        // 获取已显示的UI（用于判断界面状态）
        IUI GetShowed(int id);
        // 获取正在创建的等待删除列表
        bool IsInCreatedDels(int id);
        void AddToCreatedDels(int id);
    }

    // UIResourceHandler回调接口（可选，用于资源加载完成通知）
    public interface IUIMgrResourceHandlerCallback
    {
        void OnPackageLoaded(string pkgName, bool success);
    }
}
```

### 2. 创建 UIResourceHandler

文件：`UIResourceHandler.cs`

将 UIMgrRes.cs 中的代码迁移过来：
- UIPkgRef 类保留
- UIResourceHandler 类处理所有UI包加载/卸载逻辑
- 构造函数中初始化 YooPackage

### 3. 创建 UIStackHandler

文件：`UIStackHandler.cs`

将 UIMgrStack.cs 中的代码迁移过来：
- 栈数据：_uiStacks, _backStacks
- 构造函数注入 IUIMgrStackHandlerCallback 回调
- 关键方法：
  - `OnShowed()` - 界面显示后入栈
  - `OnHide()` - 界面隐藏前处理栈
  - `Clear()` - 清空栈

### 4. 创建 UIBlurHandler

文件：`UIBlurHandler.cs`

将 UIMgrBlur.cs 中的代码迁移过来：
- 模糊栈数据：_blurStacks
- 构造函数注入 IUIMgrBlurHandlerCallback 回调（可选，直接操作showed）
- 关键方法：
  - `OnShown()` - 界面显示时添加模糊
  - `OnHide()` - 界面隐藏时移除模糊
  - `SetSceneCamera()` - 设置场景相机

### 5. 修改 UIMgr 主类

修改 `UIMgr.cs`：
- 移除 partial class 声明
- 添加三个Handler实例作为成员变量
- 在构造函数中初始化Handler
- 保留原有的公开接口方法，内部委托给Handler处理
- 创建回调接口的内部实现类

```csharp
public partial class UIMgr : Singleton<UIMgr>
{
    // 三个独立Handler实例
    private readonly UIResourceHandler _resourceHandler;
    private readonly UIStackHandler _stackHandler;
    private readonly UIBlurHandler _blurHandler;
    
    public UIMgr()
    {
        // 初始化Handler
        _resourceHandler = new UIResourceHandler();
        _stackHandler = new UIStackHandler(new StackHandlerCallback(this));
        _blurHandler = new UIBlurHandler();
        
        // ... 原有初始化代码
    }
    
    // 回调接口实现
    private class StackHandlerCallback : IUIMgrStackHandlerCallback
    {
        private readonly UIMgr _mgr;
        public StackHandlerCallback(UIMgr mgr) { _mgr = mgr; }
        
        public void ShowByStack(int id, IUIParam param) 
        { 
            _mgr._ShowAsync<IUI>(id, param, false, false).Forget(); 
        }
        public bool IsShowing(int id) => _mgr._showing.Contains(id);
        public IUI GetShowed(int id) => _mgr._showed.TryGetValue(id, out var ui) ? ui : null;
        public bool IsInCreatedDels(int id) => _mgr._createdDels.Contains(id);
        public void AddToCreatedDels(int id) 
        { 
            if (!_mgr._createdDels.Contains(id)) _mgr._createdDels.Add(id); 
        }
    }
    
    // 代理方法（保持向后兼容）
    public async UniTask<bool> LoaUIdPackage(string[] pkgs) => await _resourceHandler.LoadUIPackage(pkgs);
    public void RemoveUIPackage(string[] pkgs) => _resourceHandler.RemoveUIPackage(pkgs);
    public void SetSceneCamera(Camera cam) => _blurHandler.SetSceneCamera(cam);
}
```

## 实施步骤

### 第一步：创建接口文件
创建 `UIHandlerInterfaces.cs`，定义所有Handler需要使用的接口：
- `IUIMgrStackHandlerCallback` - 栈处理器回调
- `IUIMgrResourceHandlerCallback` - 资源处理器回调（可选）

### 第二步：创建 UIResourceHandler
将 `UIMgrRes.cs` 代码迁移到新的 `UIResourceHandler.cs`：
- 保留 UIPkgRef 类
- 实现包加载/卸载逻辑
- 保持与原有接口一致

### 第三步：创建 UIStackHandler
将 `UIMgrStack.cs` 代码迁移到新的 `UIStackHandler.cs`：
- 实现栈管理逻辑
- 通过回调接口与UIMgr通信
- 处理界面的进栈/退栈逻辑

### 第四步：创建 UIBlurHandler
将 `UIMgrBlur.cs` 代码迁移到新的 `UIBlurHandler.cs`：
- 实现模糊效果逻辑
- 管理模糊栈
- 处理场景模糊

### 第五步：修改 UIMgr 主类
修改 `UIMgr.cs`：
- 移除 partial class 关键字
- 添加三个Handler成员变量
- 在构造函数中初始化
- 创建回调接口实现类
- 保留原有公开方法，内部委托给Handler

### 第六步：更新调用点
检查并更新 `CreateUI` 方法中的资源加载调用

### 第七步：测试验证
在 Unity Editor 中测试：
- UI显示/隐藏功能
- 栈界面进出栈
- 模糊效果
- 资源加载/卸载

## 兼容性考虑

- 在 UIMgr 中保留原有的公开接口方法（如 `LoaUIdPackage`, `SetSceneCamera`）
- 内部委托给对应的 Handler 处理
- 这样现有的调用代码无需修改
- 保持向后兼容

## 依赖注入说明

### UIStackHandler 依赖
需要通过回调接口获取以下UIMgr内部状态：
- `_showing` - 判断界面是否正在创建中
- `_showed` - 获取已显示的界面对象
- `_createdDels` - 标记需要延迟删除的界面
- `_ShowAsync` 方法 - 用于重新显示栈中的界面

### UIBlurHandler 依赖
需要直接访问以下UIMgr成员：
- `_showed` - 设置/清除界面的Filter属性
- `_mainCamera` - 设置场景模糊

为简化实现，UIBlurHandler可以直接接收IUI接口的引用，在回调中处理。
